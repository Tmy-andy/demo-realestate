Hướng Dẫn Sử Dụng Cho Chủ Đầu Tư

Trang quản trị (Owner) admin/owner.html dành cho ban quản lý dự án, đội marketing và đội kỹ thuật. Cho phép chỉnh sửa toàn bộ nội dung trang VR, xem báo cáo và quản lý leads.

Giới Thiệu Vai Trò

Tư cách Owner có toàn quyền:

  • Quản lý kho căn hộ (thêm/sửa/xoá, đổi trạng thái).
  • Xem và xử lý toàn bộ leads + booking của mọi sales.
  • Chỉnh sửa mọi nội dung trên trang VR — cảnh, hotspot, thư viện ảnh, bản đồ, tiện ích, tiến độ, pháp lý, vị trí.
  • Đổi màu chủ đạo (theme) của trang VR mà không cần code.
  • Thêm / xoá ngôn ngữ và chỉnh các chuỗi dịch.
  • Xem báo cáo Analytics.
  • Export / Import / Reset toàn bộ dữ liệu dự án.
Quan trọng — Ranh giới lưu trữ Mọi thay đổi của Owner được lưu vào localStorage của trình duyệt, không tự động đẩy lên trang VR public. Để publish, xem mục Công Bố Thay Đổi.

Hệ thống xác thực rồi tự chuyển sang owner.html. Phiên đăng nhập được lưu trong sessionStorage (mất khi đóng tab).

Đăng Xuất

Bấm nút "Đăng xuất" trên góc phải topbar. Phiên bị xoá khỏi sessionStorage và bạn được đưa về trang login.

Chú ý bảo mật Tài khoản admin có quyền chỉnh sửa mọi dữ liệu dự án. Tuyệt đối không chia sẻ mật khẩu cho người ngoài. Nên đổi mật khẩu mặc định khi triển khai thực tế (sửa mảng USERS trong admin/index.html).

Tự Động Lưu

Mọi thay đổi đều được lưu ngay vào localStorage Sau khi bấm "Lưu" trong slide panel, dữ liệu được ghi vào localStorage.ah_admin_data. Bạn có thể đóng tab và mở lại — dữ liệu vẫn còn.

Hoàn Tác?

Hệ thống không có Undo trực tiếp. Cách an toàn:

  • Trước khi chỉnh sửa lớn, hãy Export Backup tại trang Cài Đặt (xem mục 20).
  • Nếu lỡ xoá nhầm → vào Cài Đặt → Import file backup gần nhất.

Quản Lý Căn Hộ

Mỗi dòng là một căn với các trường: Mã căn · Loại · Tầng · Diện tích · Hướng · Giá · Còn/Tổng (trong loại) · Trạng thái.

  • Lọc theo loại căn (2PN / 2PN+1 / 3PN / Duplex).
  • Bấm tiêu đề cột để sắp xếp.

Thêm Ảnh — Hai Cách

Lưu Trữ & Backup

Cách Dữ Liệu Được Lưu

KeyPhạm ViNội Dung
ah_sessionsessionStorageTrạng thái đăng nhập — mất khi đóng tab
ah_admin_datalocalStorageToàn bộ dữ liệu dự án bạn đã chỉnh — tồn tại lâu dài, gắn với trình duyệt + thiết bị

Quy Trình Backup Khuyến Nghị

  1. Cuối ngày — vào Cài Đặt → Export JSON → lưu file vào folder backup (kèm ngày trong tên).
  2. Trước mỗi thay đổi lớn (đổi theme, sửa hàng loạt căn hộ) — Export thêm một bản.
  3. Upload backup lên Google Drive / cloud nội bộ để không phụ thuộc trình duyệt.
Cảnh báo Nếu xoá lịch sử trình duyệt / cache, dữ liệu localStorage sẽ mất hoàn toàn. Backup thường xuyên là bắt buộc.

Công Bố Thay Đổi Lên Trang Public

Đây là bước quan trọng nhất — biến những gì bạn chỉnh thành nội dung khách thấy.

Hiện tại CHƯA tự động Phiên bản này không có backend tự động — các thay đổi admin nằm trong localStorage, không ảnh hưởng data/project.json trên server.

Quy Trình Thủ Công

1
Vào Cài Đặt → Export JSON
Tải file JSON về máy.
2
Đổi tên file thành project.json
Phải đúng tên này để FE nhận diện được.
3
Gửi cho dev / upload lên server
Ghi đè file data/project.json trên server. Sau đó refresh trang public — thay đổi sẽ hiển thị.
4
Kiểm tra lại bằng Preview VR
Vào lại admin → bấm Preview VR để chắc chắn trang public đã cập nhật.
Hướng phát triển Khi tích hợp backend (API endpoint POST /project), nút "Publish" sẽ tự gửi JSON lên server và trang public cập nhật trong vài giây.

FAQ & Xử Lý Sự Cố

Tôi vừa chỉnh thêm 5 căn nhưng trang VR public vẫn cũ?

Bình thường — admin chỉ lưu vào localStorage. Cần làm bước Công Bố Thay Đổi (Export → upload project.json lên server).

Mở admin trên máy khác thì dữ liệu khác?

Đúng — localStorage gắn với từng trình duyệt + từng thiết bị. Để đồng bộ, sử dụng Export JSON ở máy A → Import vào máy B.

Bấm "Lưu" mà toast báo "Lỗi lưu — vượt dung lượng"?

localStorage có giới hạn ~5 MB. Thường do upload quá nhiều ảnh base64. Giải pháp: (1) xoá bớt ảnh; (2) thay base64 bằng URL CDN; (3) Export backup rồi Reset.

Mất mật khẩu admin?

Mật khẩu được hardcode trong admin/index.html (mảng USERS[]). Hỏi dev hoặc xem file trực tiếp để khôi phục.

Có giới hạn số lượng căn hộ, hotspot, POI không?

Về mặt kỹ thuật: không. Tuy nhiên hiệu năng bắt đầu giảm khi vượt 500 căn, 100 hotspot/cảnh hoặc 50 POI. Hợp lý là giữ ≤ 200 căn nổi bật trong inventory hiển thị.

Slide Panel không đóng được, kẹt ở giữa màn?

Refresh trang bằng F5. Dữ liệu chưa lưu sẽ mất — đây là lý do nên bấm Lưu ngay sau mỗi thay đổi.

Tôi muốn dùng 2 dự án khác nhau trên cùng máy?

Hiện tại admin gắn với 1 dự án (1 file project.json). Để chạy nhiều dự án, mỗi dự án phải có subdomain / folder riêng — hoặc dùng tính năng "Workspace" sau khi triển khai backend.

Có log thao tác (audit log) không?

Phiên bản hiện tại chưa có audit log. Đây là một trong các đề xuất cải tiến (xem ADMIN_IMPROVEMENT_PROPOSAL.md).
