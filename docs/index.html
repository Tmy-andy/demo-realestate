Mô Hình Dữ Liệu

Nguồn dữ liệu duy nhất Toàn bộ nội dung nằm trong data/project.json. Không có CMS, không có database.

Cấu Trúc Đối Tượng Gốc

{
  "project":          { ... },  // Thông tin dự án + amenities[] + stats[] (lồng bên trong)
  "menu":             { ... },  // 5 nhóm menu accordion
  "scenes":           [ ... ],  // Danh sách 6 cảnh panorama VR360
  "floorplan":        { units: [...] },  // Inventory căn hộ (9 căn mẫu)
  "siteMap":          { image, points: [...] },  // Bản đồ 2D + điểm bấm
  "gallery":          [ ... ],  // Thư viện ảnh (6 ảnh)
  "legal":            { developerStats, documents, banks, testimonials },
  "location":         { lat, lng, mapSrc, nearby: [...] },
  "amenitiesDetail":  { noiKhu, skyAmenity, dichVu, haTang },  // 4 tab tiện ích
  "timeline":         [ ... ]   // 8 giai đoạn thi công
}
Lưu ý về schema Key chính là project (không phải meta), menu (không phải menuGroups), floorplan.units (không phải floorplanUnits), siteMap.points (không phải siteMapPoints). amenities và stats nằm bên trong project, không phải ở gốc.

6.1 Trình Xem Panorama VR360

Module: js/vr360.js → window.VR360

Tính NăngChi Tiết
RendererThree.js WebGL, hình cầu úp ngược (r=500, 64 phân đoạn)
Nguồn panoramaCanvas sinh theo thủ tục (2048×1024)
Điều hướngKéo chuột → xoay theo kinh/vĩ độ
ZoomCon lăn chuột → FOV 40°–95°
Tự xoayBật/tắt, tạm dừng khi người dùng kéo
Toàn màn hìnhFullscreen API gốc của trình duyệt
Số cảnh6 cảnh (3 nội thất, 3 ngoại cảnh)
 
// VR360 Public API
VR360.init(container, opts)      // Khởi tạo renderer
VR360.loadScene(sceneData)       // Nạp panorama theo đối tượng cảnh
VR360.setAutoRotate(bool)        // Bật/tắt tự xoay
VR360.zoomBy(delta)              // Thay đổi FOV
VR360.projectHotspot(x, y)      // Equirect → toạ độ màn hình
VR360.getYaw()                   // Góc yaw hiện tại
VR360.lookAt(x, y)               // Hướng camera tới điểm
VR360.destroy()                  // Dọn dẹp WebGL context

Luồng Người Dùng

Luồng Chính: Tham Quan & Đặt Cọc

Tải trang └── Màn hình Loading (fetch JSON + khởi tạo Three.js) ↓ ~2–3 giây └── Trải nghiệm VR (mặc định: cảnh sky-lounge) ↓ kéo / click hotspot └── Khám phá các cảnh ↓ click "Xem Bảng Giá" └── Overlay Mặt Bằng (lọc → chọn căn → "Đặt Cọc") ↓ └── Modal đặt cọc (đã điền sẵn thông tin căn) ↓ điền họ tên + SĐT → gửi └── Màn hình thành công → CTA gọi lại

Tính Vay Vốn

Mở công cụ tính vay ↓ kéo thanh giá căn ↓ chọn % vốn tự có + thời hạn + ngân hàng ↓ kết quả cập nhật trực tiếp (trả góp/tháng · tổng lãi) ↓ xem biểu đồ trả nợ ↓ click "Tư Vấn Vay Vốn" └── Mở modal đặt cọc

Chat AI Bằng Giọng Nói

Click FAB AI → panel trượt ra ↓ hiện các bubble demo ↓ người dùng nhấn mic └── Panel → chế độ viên ↓ cấp quyền mic ↓ hiện sóng âm, bắt đầu nghe ↓ người dùng nói ↓ STT chuyển thành chữ → bubble user ↓ bot sinh phản hồi → bubble bot ↓ TTS đọc phản hồi └── tự nghe lại (lặp cho tới khi nhấn mic lại)

Đa Ngôn Ngữ (i18n)

Module: js/i18n.js → window.I18n

MãNgôn NgữCờ
viTiếng Việt (mặc định)🇻🇳
enTiếng Anh🇬🇧
zhTiếng Trung (giản thể)🇨🇳
koTiếng Hàn🇰🇷
jaTiếng Nhật🇯🇵
 
// I18n API
I18n.t('ui.bookNow', { n: 3, total: 18 })  // Dịch key tĩnh (có thể truyền biến)
I18n.tr('18 người đang xem')              // Dịch chuỗi tiếng Việt động
I18n.get()                                 // Mã ngôn ngữ hiện tại
I18n.set('en')                             // Đổi ngôn ngữ (kích hoạt sự kiện langchange)
I18n.langs()                               // Mã các ngôn ngữ có sẵn
I18n.applyStatic()                         // Dịch hàng loạt DOM theo thuộc tính data-i18n

Không Gian Tên Key

NamespacePhạm Vi
ui.*Nhãn UI chung, nút, tooltip
modal.*Modal đặt cọc — trường, validate, thành công
stepper.*Stepper mặt bằng mobile — toàn bộ bước
tour.*Nhãn các bước Help Tour

Chuỗi động (ví dụ thông báo urgency) được dịch qua I18n.tr(viString) bằng bảng tra DYNAMIC trong i18n.js. Khi đổi ngôn ngữ, sự kiện langchange được phát trên window; các component lắng nghe để render lại nội dung động.

13.3 Mô Hình Lưu Trữ

Trang admin không gọi backend. Trạng thái đăng nhập nằm trong sessionStorage; mọi dữ liệu khác phản chiếu vào localStorage.

KeyPhạm ViNội Dung
ah_sessionsessionStorage{ role, name, title, username, loginAt } — set khi login, được IIFE bảo vệ kiểm tra ở mỗi trang admin
ah_admin_datalocalStorageBản sao có thể chỉnh sửa của data/project.json — khởi tạo bằng cách fetch JSON tĩnh ở lần đầu (getFallbackData()), sau đó được mọi thao tác CRUD cập nhật trực tiếp
Lead / booking mockBộ nhớ (S.leads, S.bookings)Khởi tạo bởi getMockLeads() / getMockBookings() — không lưu khi reload
Ranh giới lưu trữ FE (index.html) luôn tải data/project.json qua mạng — các thay đổi của Owner lưu vào localStorage không phản ánh lên trang public. Muốn publish thì phải có backend hoặc bước build.
